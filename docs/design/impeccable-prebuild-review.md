# Impeccable Pre-build Review

## Verdict

前端方向通过预编码审美门槛，可以进入实现。它是任务型桌面产品，而不是营销页面；主界面应让熟悉 Linear、Notion 或 Raycast 的用户无需学习新的视觉语法。

## Confirmed Direction

- **Register:** Product UI；设计服务于监控、判断和恢复任务。
- **Color:** Restrained；纯白主工作面、青色品牌信号、琥珀仅用于爆点或预警。
- **Typography:** 单一无衬线字体体系，中文与数字共享稳定层级。
- **Composition:** 侧栏 + 连续工作区 + 按需右侧检查器；不做卡片马赛克。
- **Motion:** 150–250ms 状态反馈，只表达展开、选中、运行和完成。

## Corrections Applied

1. 将全部视觉色值改为 OKLCH，主色保持在调色种子 200° 附近。
2. 删除左侧 2px 彩色选中竖线，改为完整浅色选中面，避免常见 AI 仪表盘套路。
3. 明确首页指标采用连续横向数据带和分隔线，不生成一组等权重圆角卡片。
4. 规定所有交互组件必须覆盖默认、悬停、焦点、按下、禁用、加载、错误和成功状态。
5. 规定状态必须同时具有文字或图标，不能只靠红、绿、黄颜色区分。

## Build Gate

首个可运行页面完成后再执行正式 Impeccable critique：届时同时检查源代码、自动检测结果和 1280×800 / 1600×900 实际截图。正式复核前不得把视觉实现视为完成。
