# 抖音公开分享页解析设计

## 目标

补齐样本工具已经具备的单作品解析能力：接受常见的抖音视频链接、短链接，以及在博主页面弹窗中复制出的 `user/...?...modal_id=<视频ID>` 链接；优先从公开分享数据取得作品文案、互动数据和媒体地址，公开解析失败后才使用真实浏览器捕获，全部失败才引导用户上传本地视频。

## 输入边界

接受：

- `https://www.douyin.com/video/<纯数字ID>`
- `https://douyin.com/video/<纯数字ID>`
- `https://v.douyin.com/<非空短码>`
- `https://www.douyin.com/user/<用户标识>?modal_id=<纯数字ID>`
- `https://www.douyin.com/user/self?modal_id=<纯数字ID>`

拒绝：

- 不带 `modal_id` 的纯博主主页
- 非抖音域名、HTTP、带用户名密码、非默认端口
- 非数字、空值或多个冲突的 `modal_id`
- 把路径、查询参数或重定向目标解释成任意外部 URL

用户粘贴 `user/...?...modal_id=` 时，界面应将其识别为单作品链接，不再提示“不支持博主主页”。

## 解析顺序

### 1. 视频 ID 归一化

直接视频链接和 `modal_id` 在本地提取 ID，不发网络请求。短链接使用现有严格重定向解析器，限制跳转次数、超时和抖音域名。

### 2. `iesdouyin.com` 公开分享页

请求固定生成的地址：

`https://www.iesdouyin.com/share/video/<视频ID>`

请求使用普通移动浏览器 User-Agent，不携带应用登录 Cookie、凭据或用户浏览器资料。响应体限制大小和读取时间，只解析页面中的 `window._ROUTER_DATA` JSON；从 `video_(id)/page` 或 `note_(id)/page` 节点取得目标 ID 对应的数据。

### 3. 公开详情回退

分享页缺少目标数据时，依次请求固定抖音端点：

- `https://www.douyin.com/aweme/v1/web/aweme/detail/?aweme_id=<ID>&aid=6383`
- `https://www.douyin.com/share/video/<ID>`
- `https://www.iesdouyin.com/web/api/v2/aweme/iteminfo/?item_ids=<ID>`

每个请求使用相同的域名、重定向、超时、响应类型和体积约束。只接受响应中作品 ID 与输入 ID 完全一致的数据，不跟随媒体字段中的任意页面跳转。

### 4. 真实浏览器捕获

前三步无法得到目标作品或媒体地址时，调用已确认的 Chrome 优先、Edge 回退真实浏览器会话捕获公开 JSON 响应。登录失效或风险控制沿用真实浏览器方案的明确错误，不自动刷新或循环重试。

### 5. 本地上传回退

所有公开来源都无法取得可用媒体地址时，记录 `DOUYIN_VIDEO_DOWNLOAD_UNAVAILABLE`，作品页显示“改为上传本地视频”。如果已有标题、作者或互动数据，应保留这些元数据，不用空占位覆盖。

## 数据输出

解析器输出统一描述：

- 视频 ID
- 标题/作品文案
- 作者昵称及可用的公开作者标识
- 点赞、评论、分享等公开互动数据
- 封面 URL（可选）
- 媒体播放地址（可选）
- 数据来源：`share_router`、`detail_api`、`share_page`、`iteminfo_api` 或 `browser_capture`

媒体地址中的 `playwm` 仅在 URL 结构明确时替换为 `play`；替换后仍必须通过 HTTPS、允许的媒体主机和下载器重定向安全校验，不能把字符串替换当成可信授权。

## 与转文字流水线的关系

本阶段先复刻样本工具的“解析出媒体地址”能力，并接入现有下载/FFmpeg/转写流水线。后续可以把下载器优化为“媒体流直接输入 FFmpeg，只保留 WAV”，但该存储优化不与本解析修复捆绑，避免同时改变解析与媒体生命周期。

## 安全和可靠性

- 所有页面/API URL由应用根据纯数字 ID 固定生成，禁止用户控制请求主机。
- 只允许 HTTPS 和精确的抖音/iesdouyin 域名；重定向每一跳重新校验。
- 不导入样本工具代码，不启动或调用样本工具的 Python 服务；使用项目自身 TypeScript 实现。
- 响应设置连接、总时长和最大体积限制；取消或超时必须释放响应体。
- JSON/HTML 解析错误只进入下一回退，不把页面内容、播放地址查询参数或 Cookie 写入日志。
- 同一个视频 ID 的公开解析在单次任务内只执行一次，不并发轰炸多个端点。
- 检测到验证码、安全验证或访问频繁时停止公开端点链路，转入人工处理，不继续探测其他端点。

## 界面行为

- `modal_id` 链接通过字段同步校验，可直接点击“开始分析”。
- 任务启动后按现有异步状态展示解析、下载、转写和 AI 阶段。
- 获得公开元数据但无法获得媒体时，失败行展示真实标题和“改为上传本地视频”。
- 纯博主主页仍显示“不支持博主主页，请粘贴单条作品链接”。

## 测试与验收

自动测试覆盖：

- 五类接受输入和所有拒绝边界。
- `modal_id` 提取、短链重定向和冲突参数。
- `_ROUTER_DATA` 的 video/note 两种节点、缺失节点、畸形 JSON、超大响应。
- 四个公开来源的顺序回退、目标 ID 校验、风险控制立即停止。
- 媒体 URL 安全校验和无媒体时的本地上传错误。
- 公开解析成功时不启动浏览器；公开解析失败时只启动一次浏览器捕获。
- 用户界面接受 `user/self?modal_id=`，纯主页仍拒绝。

人工验收使用用户刚才在样本工具中成功的同类链接：确认标题、作者、互动数据和文案一致，能取得媒体时进入转写；取得不到媒体时保留元数据并提供本地上传动作。

## 非目标

- 不支持整页博主作品导入。
- 不绕过验证码、登录验证、账号/IP/设备风控或平台访问限制。
- 不依赖或复制“抖音运营工具箱”的运行文件。
- 不在本阶段改变 AI 拆解提示词、飞书同步或媒体保留期。
