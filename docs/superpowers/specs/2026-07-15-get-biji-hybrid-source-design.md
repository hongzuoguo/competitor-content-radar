# 得到大脑混合采集源设计

## 目标

用得到大脑 OpenAPI 提供稳定的博主订阅、每日作品同步和文字稿；桌面应用继续负责爆款判断、AI 拆解、作品管理与后续飞书同步。抖音浏览器采集保留为兼容模式，不再作为默认可靠链路。

## 用户流程

1. 用户在得到大脑中订阅抖音博主。
2. 在桌面应用设置页选择“得到大脑”，填写专题 ID、Client ID 和 API Key。
3. 应用每天 08:00 或用户点击“立即运行”时读取已订阅博主、作品和详情文字稿。
4. 新作品写入本地数据库；有文字稿时直接进入 AI 拆解，不下载视频、不运行本地语音识别。
5. 作品页展示同步状态、文字稿、互动指标和爆款/借鉴结论。上游未返回某项指标时显示“暂无数据”，不填造 0。

## 边界与安全

- 仅使用官方 OpenAPI；不读取浏览器 Cookie，不逆向私有接口。
- API Key 使用 Electron safeStorage 加密，渲染进程和公开设置永不回传明文。
- 官方当前公开的是只读博主接口，因此新增订阅仍在得到大脑完成；桌面应用负责同步现有订阅。
- 没有真实凭证时只能完成自动化测试，不能宣称线上链路可用。
- 点赞、评论等字段以真实 API 响应为准；缺失时保留未知状态。

## 数据与运行设计

- 新增内容源设置：`douyin_browser | get_biji`。
- 新增得到大脑公开设置：专题 ID、Client ID；密钥单独保存。
- 采集结果增加可选文字稿。运行器在文字稿存在时跳过下载和转写，直接分析。
- 博主和作品使用上游稳定 ID 去重；重复同步幂等更新指标与文字稿。
- API 错误统一映射为中文可操作提示：凭证无效、无权限、专题不存在、限流、网络失败、数据格式不兼容。

## 验收标准

- 设置页可保存内容源、专题 ID、Client ID 和加密 API Key。
- 单元测试覆盖鉴权头、分页、响应解析、错误映射、幂等同步、文字稿直达分析。
- 全量测试、类型检查和打包通过。
- 使用真实凭证同步至少 1 位博主和 1 条作品，文字稿进入 AI 分析；删除和重复同步正常。
