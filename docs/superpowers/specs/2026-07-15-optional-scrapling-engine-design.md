# 可选 Scrapling 备用采集引擎设计

## 目标

保留现有 Electron 抖音采集作为主链路。当主链路因空结果、页面结构变化或可重试错误无法取得博主作品时，应用自动按需安装一个独立的 Scrapling 备用组件，并用它采集博主资料、最近作品、点赞、评论、分享、收藏和可播放媒体地址。用户不需要自行安装 Python。

## 已确认边界

- Scrapling 不进入主安装包，避免所有用户承担体积和维护成本。
- 首次需要回退时才下载；后续复用已安装组件。
- 组件使用系统 Chrome，找不到时使用 Edge；使用应用专属浏览器目录，不读取日常浏览器资料。
- 组件只负责公开页面采集，转音频、SenseVoice 转写、AI 拆解仍走现有流水线。
- 不绕过验证码、账号/IP/设备风控。主采集器遇到风控时只允许切换到公开分享页备用引擎尝试一次；备用引擎仍遇到安全验证时立即停止并给出可执行提示，不循环刷新或重试。
- 组件不可用时继续保留“得到大脑”和“上传本地视频”两条人工兜底。

## 运行流程

1. 现有主采集器运行。
2. 若成功且有作品，直接进入现有保存与分析流水线。
3. 若返回空作品或符合回退条件的错误，组件管理器检查本地备用引擎。
4. 未安装或有兼容更新时，从固定 GitHub Release 地址获取清单和压缩包。
5. 校验协议版本、下载域名、文件大小和 SHA-256；解压到临时目录并运行健康检查。
6. 健康检查通过后原子替换当前组件，再调用 `capture_creator`。
7. 引擎以一行 JSON 输出结果，桌面应用校验后转换为现有 `Work`。
8. 若备用引擎仍失败，显示稳定中文错误与得到大脑/本地视频兜底入口。

## 组件协议

桌面应用通过标准输入发送一行 JSON：

```json
{"protocolVersion":1,"command":"capture_creator","creatorId":"...","profileUrl":"https://...","profileDirectory":"..."}
```

组件标准输出只允许一行 JSON：

```json
{"protocolVersion":1,"ok":true,"creator":{"name":"...","profileUrl":"..."},"works":[...]}
```

标准错误仅用于脱敏诊断。进程超时、退出码异常、非法 JSON、协议版本不匹配或结果字段不合法都视为组件失败。

## 下载与安全

- 清单地址由应用常量固定，必须是 HTTPS。
- 资源最终地址只允许 GitHub 官方 Release 域名。
- SHA-256 必须完全匹配，压缩包大小设上限。
- 解压时拒绝绝对路径、`..` 和目录穿越。
- 安装位置为用户数据目录下的 `components/scrapling/<version>`。
- 先下载到临时目录，健康检查成功后再更新 `current.json`，旧版本保留到替换成功。
- 不执行清单提供的任意命令；可执行文件名和启动参数由应用固定。

## 发布形式

备用组件使用 Windows x64 one-directory ZIP，内含 PyInstaller 构建的引擎和运行库，但不包含 Chrome/Edge。独立工作流发布组件 ZIP 与清单，使组件可以独立于桌面应用升级。

## 状态与错误

内部状态：`not_installed`、`downloading`、`installing`、`ready`、`error`。本阶段只增加必要的进度/错误通知，不新增复杂设置页。

稳定错误包括：清单不可用、下载失败、校验失败、浏览器未找到、组件启动失败、协议错误、采集超时、抖音风控。所有自动重试都有次数和超时上限。

## 验收标准

- 单元测试覆盖清单校验、域名限制、SHA-256、原子安装、协议校验、超时和回退条件。
- Python 引擎可被 PyInstaller 构建为无需本机 Python 的 ZIP。
- 打包后的引擎能用用户提供的博主名片获得作品列表与互动数据。
- 桌面应用在主链路失败时可自动调用本地已安装组件。
- 发布组件后，在没有 Python 的全新用户目录中完成“名片链接 → 自动下载组件 → 作品列表与指标”的端到端验证。
- 全量测试、类型检查和桌面构建通过后才声明完成。
