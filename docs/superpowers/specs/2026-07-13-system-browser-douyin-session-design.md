# 抖音真实浏览器会话设计

## 目标

将抖音登录和采集从 Electron 内嵌窗口迁移到电脑已安装的真实浏览器，降低内嵌浏览器环境被抖音判定为异常访问的概率，并让首次扫码后的登录状态可持续复用。

## 已确认方案

应用优先使用 Google Chrome；未安装 Chrome 时自动使用 Microsoft Edge。两者都不可用时，明确提示用户安装其中一个浏览器，不回退到 Electron 内嵌登录窗口。

应用使用独立的浏览器用户数据目录，不读取或修改用户日常浏览器的个人配置、Cookie、扩展和浏览历史。

## 用户流程

1. 用户点击“登录抖音”。
2. 应用启动真实 Chrome；Chrome 不存在时启动 Edge。
3. 浏览器使用应用专属配置目录打开抖音首页。
4. 用户使用抖音 App 扫码。
5. 应用通过页面状态和抖音会话 Cookie 判断登录成功，并在桌面应用中显示“抖音已连接”。
6. 用户可关闭浏览器窗口；专属会话保留。
7. 后续博主采集和单作品采集复用同一个浏览器类型和配置目录，无需重复登录。

## 架构边界

### 浏览器发现

浏览器启动器只负责按已知 Windows 安装位置和系统注册信息寻找 Chrome、Edge，并返回可执行文件路径与浏览器类型。优先级固定为 Chrome、Edge。

### 专属会话

浏览器配置保存在应用用户数据目录下的独立子目录。应用不得连接用户正在使用的默认浏览器配置，也不得复制、解密或导入用户日常浏览器 Cookie。

同一时间只允许一个抖音浏览器会话使用该目录。登录与采集请求通过会话管理器串行化，避免配置锁冲突。

### 浏览器控制

应用以受控调试端口启动浏览器，并通过 Chrome DevTools Protocol 连接。端口只绑定本机回环地址，生命周期随会话结束；应用不对局域网或公网开放调试端口。

登录窗口保持可见。后台采集允许使用同一真实浏览器的受控窗口，但不得使用 Electron 内嵌窗口作为隐式回退。

### 登录判定

登录成功必须由明确证据确认：抖音页面进入已登录状态，并存在预期的会话标识。仅凭页面加载完成或窗口被关闭不能判定成功。

用户在未登录时关闭窗口，应用保持“未连接”，并提示重新打开登录窗口。

## 采集数据流

采集器通过真实浏览器会话打开博主主页或单视频标准链接，监听目标抖音 JSON 响应，并沿用现有解析、去重和后续导入流水线。浏览器启动、连接、页面加载、响应捕获和关闭分别设置明确超时与稳定错误码。

浏览器进程意外退出时，当前任务进入可重试失败；下次重试重新启动同一专属会话，不删除已保存的登录状态。

## 风控处理

检测到“访问太频繁”、验证码、人机验证或安全验证后：

- 立即停止当前自动采集，不循环刷新或自动重试。
- 保持浏览器可见，允许用户查看抖音原始提示。
- 在应用中显示“抖音限制了当前访问，请稍后在登录窗口完成验证”，并记录稳定错误码。
- 自动任务进入需要人工处理状态；只有用户明确重试或下一次计划运行才再次访问。

本方案不承诺消除抖音账号、IP 或设备侧风控，也不实现指纹伪装、验证码绕过或其他规避平台安全机制的行为。

## 错误与恢复

- 未找到浏览器：提示安装 Chrome 或 Edge。
- 浏览器启动失败：显示可重试错误，不回退内嵌窗口。
- 调试连接失败：终止本次会话并清理子进程。
- 未登录关闭：保持未连接，允许再次登录。
- 会话过期：自动采集停止并引导重新扫码。
- 风控：停止自动重试，保留原始验证页面供用户处理。
- 应用退出：先停止新任务，再关闭受控浏览器和调试连接。

所有用户提示使用中文，不暴露 CDP、Cookie 名称、调试端口等内部技术细节。

## 安全与隐私

- 浏览器数据目录仅供本应用使用。
- 日志不得写入 Cookie 值、授权头、手机号、二维码内容或完整敏感 URL 参数。
- 调试端口仅绑定 `127.0.0.1`，并使用动态空闲端口。
- 应用只读取判断登录和采集所需的最少会话信息。
- 卸载或清理登录数据必须由用户主动触发，不能在普通失败重试时删除。

## 测试与验收

自动测试覆盖：

- Chrome 优先、Edge 回退和均未安装。
- 专属配置目录与用户默认配置隔离。
- 登录成功、未登录关闭、会话过期。
- 同一配置目录的会话串行化。
- 调试端口仅绑定本机并在结束时释放。
- 风控后不自动刷新或循环重试。
- 浏览器崩溃、连接超时和应用退出时清理资源。
- 日志脱敏和稳定错误码。

人工验收必须在 Windows 安装包中完成：首次扫码成功、关闭并重启应用后仍保持登录、能够采集一个博主主页、遇到风控时停止访问并给出可执行提示。

## 非目标

- 不接管用户日常 Chrome 或 Edge 配置。
- 不导入用户默认浏览器 Cookie。
- 不绕过验证码、设备验证或平台访问限制。
- 不支持 Firefox 或其他浏览器。
- 不在本阶段改变转写、AI 分析、飞书同步或作品列表界面。
